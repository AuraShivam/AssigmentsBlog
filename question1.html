â“ Question

There is an integer array nums sorted in non-decreasing order (not necessarily with distinct values).
Before being passed to your function, nums is rotated at an unknown pivot index k (0 <= k < nums.length)...
Given the array nums after the rotation and an integer target, return true if target is in nums, or false if it is not in nums.
You must decrease the overall operation steps as much as possible. 

Example 1:
Input: nums = [2,5,6,0,0,1,2], target = 0
Output: true

Example 2:
Input: nums = [2,5,6,0,0,1,2], target = 3
Output: false

Constraints:
1 <= nums.length <= 5000
-104 <= nums[i], target <= 104

ğŸ’¡ Intuition

Use Binary Search. If nums[left] == nums[mid] == nums[right] (duplicate scenario), move left and right pointers inwards. Otherwise, check which half is sorted, then reduce the search space.

ğŸ” Approach

  1. If target == nums[mid], return true.
  2. If nums[left] == nums[mid] == nums[right], increment left and decrement right to shrink bounds.
  3. If the left half is sorted and target is in it, search left; else search right.
    If right half sorted, and target is in it, search right; else search left.
Time Complexity: Average O(log n), worst O(n) with many duplicates.
Space Complexity: O(1)
ğŸ’» C++ Solution
class Solution {
public:
    bool search(vector<int>& nums, int target) {
        int left = 0, right = nums.size() - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) return true;
            if (nums[left] == nums[mid] && nums[mid] == nums[right]) {
                ++left;
                --right;
            } else if (nums[left] <= nums[mid]) {
                if (nums[left] <= target && target < nums[mid]) right = mid - 1;
                else left = mid + 1;
            } else {
                if (nums[mid] < target && target <= nums[right]) left = mid + 1;
                else right = mid - 1;
            }
        }
        return false;
    }
};
